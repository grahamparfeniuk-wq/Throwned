# Product scope

Use this folder for **scoped** product detail: what ships in a phase, acceptance criteria, and explicit non-goals.

Suggested files (add as needed):

- `phase-1.md` — shipped or frozen scope
- `phase-2.md` — current milestone
- `non-goals.md` — explicit deferrals to avoid scope creep

Keep top-level `docs/*.md` files high-level; put granular checklists and tickets-to-be here.
